
# 🧠 Development Method:

## discuss → plan → execute → test

This is the workflow we will use to build software.

> The goal is not just to write code —  
> but to **think, structure, and improve like a developer**.

---

# 🔁 The Loop (Core Idea)

You will repeat this process:

```
discuss → plan → execute → test → repeat
```

If something breaks → go back  
If you want to improve → go back

---

# 1. 🧠 Discuss (Understand the Problem)

## 🎯 Goal

Understand **what you are building before coding**

## Ask yourself:

- What is the problem?
- What should the program do?
- What is input → output?
- What is the simplest version (MVP)?

## Example:

> “We want a program that stores and shows tasks”

## ✅ Output:

- A short description of the system
    
- Clear goal
    

---

# 2. 🗺️ Plan (Structure Before Code)

## 🎯 Goal

Break the problem into **small, clear steps**

## Use:

get-shit-done

## What you do:

- Create tasks
- Split into smaller subtasks
- Decide order
- Define requirements (clarify input, output, edge cases)
- Plan tests (what to verify and how to test each feature)
    

## Example plan:

- Setup project
    
- Create data structure
    
- Add task
    
- List tasks
    
- Mark task as done
    

## ✅ Output:

- A clear task list (no code yet)
    

---

# 3. ⚡ Execute (Build Step by Step)

## 🎯 Goal

Implement **one task at a time**

## Rules:

- Do NOT jump ahead
    
- Keep it simple
    
- Run code often
    

## Focus:

> “Make this one thing work”

## ✅ Output:

- A working part of the system
    

---

# 4. 🧪 Test (Verify)

## 🎯 Goal

Check if it actually works

## Do:

- Run the program
    
- Try normal use
    
- Try edge cases
    

## Ask:

- Does it work as expected?
    
- What is broken?
    
- What can be improved?
    

## ✅ Output:

- Working feature OR new tasks
    

---

# 🔁 Repeat (Important)

After testing:

### If something is broken:

→ go back to **plan**

### If you want to improve:

→ go back to **discuss**

---

# ⚠️ Key Principles

## 1. Think before coding

Do not jump straight into code.

---

## 2. Work in small steps

Big problems become easy when broken down.

---

## 3. One task at a time

Avoid chaos.

---

## 4. Test often

Catch problems early.

---

## 5. Loop continuously

Good software is built through iteration.

---

# 🚀 Why This Matters

This is how real developers work.

You are learning:

- Problem solving
    
- System thinking
    
- Debugging
    
- Structured development
    

---

# 💡 Simple Rule to Remember

> If you are stuck:
> 
> ❌ Don’t write more code  
> ✅ Go back to the previous step
