
# ⚙️ Tool: Get ~~Shit~~ Done (GSD)

## What is GSD?

get-shit-done is a simple tool for organizing work.

It helps you:

- Break problems into tasks
    
- Keep track of what to do
    
- Work step-by-step like a developer
    

> Think of it as your **task brain** while coding

---

# 🎯 Why we use GSD

When building software, it’s easy to:

- Get lost
- Jump between ideas
- Forget what to do next 

GSD solves this by giving you:

- Clear structure
- Focus
- A plan to follow

---

# 🔁 How GSD fits the method

## Our workflow:

```
discuss → plan → execute → test
```

---

## 🧠 Discuss

You define:

- What you are building
- What the goal is


---

## 🗺️ Plan (GSD starts here)

You use GSD to:

- Create tasks
- Break tasks into smaller steps
- Decide what to do first

### Example tasks:

- Setup project
- Create data file
- Add task
- List tasks

👉 GSD becomes your **source of truth**

---

## ⚡ Execute

You:

- Pick ONE task from GSD
- Implement it

### Important:

- Only work on the current task
- Do not jump ahead

👉 GSD tells you:

> “This is what you should build now”

---

## 🧪 Test

After building:

- Test your code
- Check if it works

If something is wrong:

- Add a new task in GSD

If everything works:

- Move to the next task

---

# 🔁 Loop with GSD

```id="8q3k2d"
GSD task → build → test → update GSD → repeat
```

GSD is always updated as you learn more.

---

# ⚠️ Rules when using GSD

- Always start with a plan
    
- Keep tasks small
    
- Only do one task at a time
    
- Update tasks when things change
    

---

# 💡 Example

## In GSD:

```
[ ] Add task feature
    - Read tasks.json
    - Add new task
    - Save file
```

## You do:

- One step at a time
    
- Test after each step
    

---

# 🚀 Goal

By using GSD, you learn to:

- Think before coding
- Work step-by-step
- Stay organized
- Build like a real developer

---

# 🧠 Simple Rule

> If you don’t know what to do next:
> 
> 👉 Look at GSD, not your code
